| Plateforme                | Documentation officielle pour créer / récupérer vos identifiants                                                                                                                             |
| ------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Facebook (Page)**       | • Générer un **User/Page Access Token** via le guide Facebook Login ([Développeurs Facebook][1]) <br>• Trouver le **Page ID** depuis l’interface Business ou via l’aide Meta ([Facebook][2]) |
| **Instagram (Graph API)** | Convertir un **short‑lived token** en **long‑lived token** et récupérer l’`instagram_user_id` ([Développeurs Facebook][3])                                                                   |
| **Pinterest**             | Tutoriel OAuth 2 pour obtenir un **access\_token** après consentement utilisateur ([Pinterest Developers][4])                                                                                |
| **Google My Business**    | Implémenter OAuth 2.0 pour obtenir un **access\_token** et lier vos `account_id` / `location_id` ([Google for Developers][5])                                                                |
| **X / Twitter v2**        | Générer un **Bearer Token** (app‑only ou user context) via le guide Auth basics ([Developer X][6])                                                                                           |
| **TikTok for Developers** | Gestion complète des **user access tokens** (fetch, refresh, revoke) via OAuth 2 ([TikTok pour les développeurs][7])                                                                         |
| **PrestaShop Webservice** | Créer une **Webservice API Key** depuis le back‑office (« Ajouter une clé webservice ») ([PrestaShop][8])                                                                                    |
| **Brevo / Sendinblue**    | Générer une **API Key v3** depuis « SMTP & API » dans votre profil Brevo ([developers.brevo.com][9])                                                                                         |
